Laravel theme — hand-picked vs. 3-seed OKLCH, two base bgs

Left column is hand-tuned to match the real Laravel docs (the target). The middle and right columns are generated from just 3 seeds — red primary, green secondary, and one base bg — differing only in that base bg (#222221 vs #171717). Every swatch is computed live in OKLCH; nothing here is hand-typed CSS except the reference column.

Brand reference (from the Laravel docs): #FF2D20 red keyword/inline-code  ·  green strings  ·  warm near-neutral dark chrome (#222221), muted grey comments.

What the comparison shows

  • B — Hand-picked (reference). ~22 individually tuned hexes matched to the docs screenshot: bg #222221, red keywords, green strings. This is the target — but every theme costs this much effort and the contrast guarantees live in the author's head.
  • C — OKLCH from 3 seeds. The entire palette comes from red primary + green secondary + one base bg. The neutral backbone (sidebar, active line, dock, border, gutter, comment, fg) is the base bg's own hue + chroma, L-ramped in OKLab. Status/selection/active-line and the contrast text are all derived. It lands within touching distance of the hand-tuned column.
  • Base bg is the one knob that matters. Middle is #222221, right is #171717 — only that single seed changed. Everything else re-ramps off it automatically, so picking the bg darkness is the whole authoring decision for the neutral chrome.
